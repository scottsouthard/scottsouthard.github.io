Enumerable Method "Map" (vs. "Each")

The Enumerable module is a great example of Ruby's versatility and adaptability. Any class of objects with the Enumerable module attached can perform any of the many tasks assigned to Enumerable. he tasks span a wide array (heh) of iterators, each stemming from the seminal each iterator.
map is a very close relative to each, in that it iterates through every element in a collection and then performs a task dictated in a block of code that follows. Whatever that code is isn't all that important, but what they return from that code makes all the difference.
each iterates through every element of a collection, performs a task, and returns the selected elements in their original form. The more transformative map similarly iterates through every element, performing the dictated action, but returns the modified elements.
Let's look at the two as they work on an array:

2.3.0 :003 > arr.each { |num| num + 1}
=> [1, 2, 3, 4, 5, 6]
2.3.0 :004 > arr.map { |num| num + 1}
=> [2, 3, 4, 5, 6, 7]

You can see each iterates through the code and returns the originals while map returns the transformed results. While this instance seems like a silly test we'd never perform, let's try another:

2.3.0 :010 > arr.each { |num| num.even? }
=> [1, 2, 3, 4, 5, 6]
2.3.0 :011 > arr.map { |num| num.even? }
=> [false, true, false, true, false, true]

In this instance, we can see the useless return of each. It went through each element, determined if each was an even number or not, but then neglected to tell us! Whereas map was far more helpful, iterating through every element and returning a boolean value of true or false, depending on the even-ness of each number.

Both of these iterators have important uses. Sometimes you don't want to change the originals, just to look through them. Other times, we need to make big changes and see the results!
