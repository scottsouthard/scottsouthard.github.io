Ruby Classes

Every time Ruby is put in use, Ruby classes are also put in use. They're the building blocks of the language and they define every object (of the object oriented programming variety) we encounter.
A class is a heirarchical tool, giving traits to every object created within the class. So every integer can be added or multiplied, while every string can be split apart.

Let's take a look at a new class of our own to get a better idea of how this works:


Here we have a new class called Jeans (and for the sake of brevity, we're assuming they're men's jeans). We've taken two main traits of any pair of jeans (the size of the waist and the length of the legs) and made them instance variables.

Instance variables are the veins of any class, as their scope is larger than just one method. In fact, instance variables are able to remain constant throughout an object, available to the whims of any method called on them.



Here we see that after calling thanksgiving on 511s (our original instance of jeans), we can alter the value of @waist even though we never defined it in our method. How cool is that?

Classes control the basics of every object that comes into play in every instance of Ruby. Whether we see them or not, they're dictating the variables and abilities of all the tools we work with. Whether creating our own or using the built-in classes of Ruby, the heirarchy of classes passes down traits and we work with them one way or another.
